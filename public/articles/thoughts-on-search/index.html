Searching and APIs

  • APIs that are a subset of search

    Often an API is a subset of search. Anytime a user of the API gets a list of items back matching some criteria, it might simply be a search query masquerading as an API call. One can turn search queries into an API call. For example, give me all objects that were created on day X (date: xx/yy/zzzz) could be 
    List<ObjectType> getObjectsCreatedOn(Date someDay)
    So we know that some API calls have a corresponding search query.

  • One search API vs many equivalent API calls

    One can therefore reduce many equivalent API calls to a single search API. One still needs typing for the search result objects and potentially transforming objects from a serialized form to a reiffied form (e.g. bytes -> plain old objects).
